一、命名规范
1. 通用原则
所有命名需语义化、无歧义，优先使用英文（避免拼音 / 中英文混用），长度适中，能直观体现变量 / 函数 / 文件的用途，禁止使用无意义字符（如a1、tmp（非临时变量））。
2. 变量命名
采用 “小写字母 + 下划线” 的蛇形命名法（Python / 后端），如user_input、dp_history、best_value；
前端 JavaScript 变量可采用小驼峰命名法，如groupNum、scatterImg；
常量（不可修改的值）采用 “全大写 + 下划线”，如ANIMATION_INTERVAL、MAX_CAPACITY、SCATTER_SIZE；
布尔型变量前缀加is/has/can，如is_valid、has_data、can_solve。
3. 函数 / 方法命名
采用 “动作 + 对象” 的蛇形命名法（后端）/ 小驼峰（前端），如read_data、draw_scatter_plot、export_report（后端），uploadFile、playAnimation（前端）；
函数名需直接体现核心功能，避免抽象命名（如用calculate_best_value而非compute）。
4. 文件 / 目录命名
采用小写字母 + 下划线，如data_processor.py、scatter_plot_utils.py；
前端文件可按功能命名，如index.html、animation.js、style.css；
避免特殊字符、空格，禁止中文命名。
二、注释规范
1. 注释原则
注释需 “简洁、准确、必要”，说明 “为什么做” 和 “做什么”，而非重复代码逻辑；核心函数、复杂逻辑、异常处理必须加注释，简单逻辑（如变量赋值）可省略。
2. 函数 / 方法注释
采用文档字符串（Docstring）格式，包含：功能描述、参数说明（类型、含义）、返回值说明（类型、含义）、异常说明（可选）；
3. 代码行注释
复杂逻辑、关键步骤添加单行注释，注释位于代码上方或右侧，使用#（Python）、//（JS/CSS）；
4. 注释格式
注释语言统一使用中文（面向中文开发团队）；
避免注释与代码脱节（代码修改后同步更新注释）；
禁止使用无意义注释（如# 循环、// 赋值）。
五、代码编写规范
1. 语法与格式
缩进统一：Python 使用 4 个空格，前端使用 2 个空格 / 4 个空格（统一即可），禁止使用制表符（Tab）；
行长度限制：单行代码不超过 80-120 个字符，过长代码换行（Python 用\或括号，JS 用+或模板字符串）；
空行分隔：函数之间、逻辑块之间用空行分隔，提升可读性；
运算符 / 标点：运算符两侧加空格（如a = b + c），逗号后加空格（如list = [1, 2, 3]）。
2. 逻辑编写原则
单一职责：一个函数 / 方法仅实现一个核心功能，避免 “大函数”（超过 50 行需拆分）；
避免冗余：重复逻辑提取为通用函数（如多次用到的绘图样式设置、数据校验）；
提前返回：函数中先校验入参合法性，不满足则提前返回，减少嵌套层级；
减少嵌套：嵌套层级不超过 3 层，超过则拆分函数。
3. 异常处理规范
全覆盖：文件操作、网络请求、输入输出、数据转换等易出错场景必须加异常捕获；
明确性：捕获具体异常（如FileNotFoundError、ValueError），避免直接捕获Exception；
友好处理：捕获异常后打印日志（便于定位问题）+ 返回友好提示（用户 / 调用方可见），不吞异常；
4. 输入校验规范
前端 + 后端双重校验：前端校验（如输入是否为数字、范围是否合法）提升用户体验，后端校验保障数据安全；
校验规则：非空校验、类型校验、范围校验、格式校验（如文件格式、字符串格式）；
